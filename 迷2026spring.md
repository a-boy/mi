The Game of Life, also known as Conway's Game of Life or simply Life, is a cellular automaton devised by the British mathematician John Horton Conway in 1970.[1] It is a zero-player game,[2][3] meaning that its evolution is determined by its initial state, requiring no further input. 

At each step in time, the following transitions occur:

Any live cell with fewer than two live neighbours dies, as if by underpopulation.
Any live cell with two or three live neighbours lives on to the next generation.
Any live cell with more than three live neighbours dies, as if by overpopulation.
Any dead cell with exactly three live neighbours becomes a live cell, as if by reproduction.
The initial pattern constitutes the seed of the system. The first generation is created by applying the above rules simultaneously to every cell in the seed, live or dead; births and deaths occur simultaneously, and the discrete moment at which this happens is sometimes called a tick.[nb 1] Each generation is a pure function of the preceding one. The rules continue to be applied repeatedly to create further generations.
Cellular automata have their origins in the work of Stanisław Ulam and John von Neumann in the 1940s. Development continued through the 1950's and 1960's when, in 1968, John Horton Conway began doing experiments with a variety of different two-dimensional cellular automaton rules. Conway's initial goal was to define an interesting and unpredictable cellular automaton.[3] According to Martin Gardner, Conway experimented with different rules, aiming for a ruleset that would allow for patterns to "apparently" grow without limit, while keeping it difficult to prove that any given pattern would do so.

ComfyUI 是一个基于节点（Node-based）的图形化用户界面，专为 Stable Diffusion 等扩散模型设计。它允许用户通过连接不同的“功能块”（节点）来构建复杂的图像、视频、音频及 3D 生成工作流，而无需编写代码。 [1, 2, 3, 4] 
核心特性

* 节点化设计：采用类似电路板的连线方式，将加载模型、输入提示词、采样、解码等步骤拆分为独立节点，实现对生成过程的精确控制。
* 本地运行：主要设计用于在本地 GPU 上运行（支持 NVIDIA, AMD, Intel, Apple Silicon），无订阅费，且数据隐私性高。
* 极高灵活性：支持复杂的自定义节点（Custom Nodes），可以集成各种最新的 AI 技术（如 ControlNet, IP-Adapter, 各种视频生成模型）。
* 资源占用低：相比其他界面（如 Automatic1111），ComfyUI 通常能更高效地利用显存，适合在显存较小的设备上运行大型模型。 [1, 2, 5, 6, 7, 8, 9] 

如何开始

   1. 官方源码/下载：访问 [GitHub 仓库](https://github.com/comfy-org/ComfyUI) 获取最新代码或下载 Windows 便携包 以快速启动。
   2. 管理工具：强烈建议安装 [ComfyUI-Manager](https://github.com/Comfy-Org/ComfyUI-Manager)，用于一键安装缺失节点和更新插件。
   3. 学习资源：可以通过 [ComfyUI Wiki](https://comfyui-wiki.com/en) 或 官方文档 学习基础操作。
   4. 云端版本：如果没有高性能显卡，可以使用 [Comfy Cloud](https://www.comfy.org/cloud) 或 [ComfyICU](https://comfy.icu/) 等付费云端平台。 [1, 8, 10, 11, 12, 13, 14]

## Claude 重磅更新：Computer Use + Dispatch 双功能深度解析
### 一、核心更新概览 
Claude于2026年3月推出**Computer Use（电脑视觉操控）** 与**Dispatch（手机远程调度）** 双功能组合，标志AI智能体进入**视觉操控时代**。无需依赖API，通过屏幕识别模拟真人操作，可直接操控微信、剪映等软件，支持手机远程指令，对标并超越OpenClaw（小龙虾）。
### 二、Computer Use：纯视觉电脑操控 
- **工作原理**：截图→识别→决策→执行，通过HID协议模拟鼠标键盘操作，无需系统权限。 
- **核心优势**：**无API限制**，像真人一样操作任意桌面软件（如微信、剪映）。 
- **典型应用**：一句话发文件、刷群聊、跨软件数据迁移、自动填写表单等。
### 三、Dispatch：手机远程任务调度 
- **定位**：手机与电脑建立**持久化对话通道**，实现“手机下单，电脑执行”。 
- **连接方式**：HiPC扫码配对，支持开机自启动、HID输入、提示音/震动反馈。 
- **功能**：远程指令、定时任务（如清晨整理邮件）、多任务分发（不污染上下文）、跨设备同步。 
### 四、安全防护机制 
采用**三重防护**：每步授权（敏感操作需许可）、应用黑名单（默认禁止银行等敏感软件）、防注入防护（检测恶意指令），权限最小化。
### 五、与OpenClaw对比 
| 维度 | Claude（双功能） | OpenClaw（小龙虾） | 
|--------------|---------------------------------|-----------------------------| 
| 技术归属 | Anthropic官方原生，深度整合大模型 | 第三方开源，模型无关 | 
| 部署方式 | 云端+官方App，一键配对 | 完全自托管，需本地配置 | 
| 安全机制 | 内置多层防护 | 需用户自行配置 | 
### 六、使用门槛 
- **平台**：仅支持**macOS**，研究预览阶段，成功率约50%。 
- **订阅**：需Claude Pro（$17/月）或Max（$100-200/月）。 
- **步骤**：更新Claude桌面端→开启Computer Use→手机扫码配对Dispatch→输入指令。
### 七、应用场景 
- **移动办公**：通勤时指令电脑整理聊天记录、剪辑视频。 
- **自动化工作流**：内容创作者远程剪辑发布视频；职场人士定时整理邮件、生成周报。 
- **跨设备协作**：手机拍照上传，电脑AI自动OCR识别归档。

## turn Chrome's address bar into a notepad with one line of text 
```
data:text/html, <html contenteditable>
```

The code `data:text/html, <html contenteditable>` is a "data URI" trick that instantly transforms your web browser tab into a simple, functional text editor.   
### How it Works
- `data:text/html,`: This prefix tells the browser to treat the following text as raw HTML code rather than a website address.
- `<html contenteditable>`: This is a standard HTML attribute that makes any element (in this case, the entire page) editable by the user.   
### Key Features
- Rich Text Support: Most modern browsers allow basic formatting via keyboard shortcuts, such as Ctrl+B for bold, Ctrl+I for italics, and Ctrl+U for underline.
- No Setup Required: It works offline and requires no internet connection or external software.
Lightweight: It is often referred to as a "one-line notepad" or "scratchpad" for quickly jotting down notes or temporary snippets. 
### Limitations to Consider
- No Auto-Save: Your text is not saved automatically. If you refresh the page or close the tab, your content will be lost.
Saving Manually: To keep your work, you must manually save the page as an HTML file (e.g., Ctrl+S) or copy the text to another application.
- Privacy: Because it runs entirely in your local browser instance without external scripts, it is generally considered private for temporary notes. 

For a more advanced version with built-in styling, some users prefer a slightly longer version:
```
data:text/html, <title>Notepad</title><body contenteditable style="font-size:1.5rem;line-height:1.5;padding:10%;max-width:800px;margin:auto;">
```

# Find files over 10 MB, use Copilot CLI to describe them, and email a summary

EMAIL_TO="user@example.com"
SUBJECT="Large file found"
BODY=""

while IFS= read -r -d '' file; do
    size=$(du -h "$file" | cut -f1)
    description=$(copilot -p "Describe this file briefly: $file" -s 2>/dev/null)
    BODY+="File: $file"$'\n'"Size: $size"$'\n'"Description:     $description"$'\n\n'
done < <(find . -type f -size +10M -print0)

if [ -z "$BODY" ]; then
    echo "No files over 10MB found."
    exit 0
fi

echo -e "To: $EMAIL_TO\nSubject: $SUBJECT\n\n$BODY" | sendmail "$EMAIL_TO"
echo "Email sent to $EMAIL_TO with large file details."
